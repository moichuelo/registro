Module: verifyAdmin

Middleware: verifyAdmin ----------------------- Verifica que el usuario esté autenticado mediante un JWT válido en la cookie "token" y que su rol sea "admin". Flujo: - Extrae el token JWT de `req.cookies.token`. - Si no hay token, responde con `403 Acceso denegado`. - Verifica la validez del token usando la clave secreta del entorno (`process.env.JWT_SECRET`). - Si el token es válido pero el rol del usuario no es "admin", responde con `403 Acceso solo para administradores`. - Si todo es correcto, agrega el objeto `req.user` con el payload decodificado y continúa.
